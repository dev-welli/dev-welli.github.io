Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

Just like the Udacity video describe, if we use the analogy of building a house:

  • HTML is what the house made of, its structure such as walls, doors, windowss, roof etc
  • CSS is the design of the house, what it will be looks like, what colours, what shape
  • DOM is how the house put together. Where that door/window will be connected to or part of, this wall or that wall?
What is meant by boxifying design?

The process where you breakdown a webpage design into boxes, big and small. This will make it easier to design the website in a logical way

What is the box model?

A layout of page can be represented in boxes, large and small. One or more boxes can be inside another. With CSS you can set its size, potion, and properties such as margins, borders, padding
